🔄
沙箱状态机
确定性状态转换
  • Pending — 请求已接受，正在创建
  • Running — 正常运行，可执行操作
  • Pausing — 正在暂停，保存状态
  • Paused — 已暂停，Pod 已释放
  • Resuming — 正在恢复，重建 Pod
  • Stopping / Terminated — 终止中 / 已终止
  • Failed — 创建或操作失败

🔥 Pool 预热机制

预创建 Pod 池，亚秒级沙箱分配，自动扩缩容

♨️
容量温度控制
四参数自动调节
  • BufferMin / BufferMax — 空闲缓冲窗口
  • PoolMin / PoolMax — 池绝对上下限
  • 回中策略 — 缓冲超范围时回到 (min+max)/2
  • 滞回 (Hysteresis) — 避免边界震荡
  • maxUnavailable (25%) — 扩容速率限制
  • 未就绪 Pod 计入 — 防过度扩容
🎯
多级智能选池
Predicate + Scorer 链
  • Capacity — PoolMax - Allocated ≥ replicas
  • Image — 镜像交集匹配
  • LabelSelector — 自定义标签匹配 (可配)
  • NodeSelector — 节点亲和性兼容
  • Resource — CPU/Memory 资源充足
  • Scorer — LeastAllocated / MostAllocated 打分
♻️
调度与回收
注解驱动 · 三阶段事务
  • PackedSchedule — 贪心按序分配
  • SpreadSchedule — 分散分配
  • 三阶段释放 — alloc-status → alloc-release → alloc-released
  • Recycle 策略 — Delete · Restart · Noop
  • 逐出保护 — 已分配 Pod 不受节点维护影响
  • 孤儿 GC — force-delete 沙箱 Pod 自动回收
  • 滚动更新 — 模板变更 maxUnavailable 速率

📸 快照 (Pause / Resume)

声明式快照 — 暂停时保存 rootfs 到 OCI Registry，恢复时重建

💾
快照 CRD 与状态机
SandboxSnapshot
  • 极简 spec — 仅需 sandboxName
  • 状态机 — Pending → Committing → Succeed / Failed
  • Per-Container — 多容器独立快照 + digest 记录
  • OwnerRef 链 — BatchSandbox → Snapshot → Commit Job
  • 标签生成 — snap-gen{generation} (内部) / snap-{sha256} (公开)
🏗️
Image Committer
nerdctl 六步流程
  • Step 1 — 定位容器 (K8s label 过滤)
  • Step 2 — nerdctl pause (保证 FS 一致性)
  • Step 3 — nerdctl commit (rootfs → OCI)
  • Step 4 — nerdctl unpause (无论成败都恢复)
  • Step 5 — nerdctl login + push
  • Step 6 — nerdctl inspect 提取 digest
10min 超时Backoff=3信号安全
🔄
Pause/Resume 流程
5-Case 状态调度
  • handlePause — 停止任务 → 创建 Snapshot
  • syncPauseOrClear — 轮询快照 → completePause
  • completePause — 删除 Pod / 脱离 Pool
  • handleResume — ACK Resuming 阶段
  • continueResume — 替换镜像 → 重建 Pod
  • Retry Bridge — Server 端 nil→target 两步桥接
