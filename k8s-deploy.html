Quick Start

🚀 快速开始

选择最适合你的安装方式

Shell 
# 从 GitHub Release 安装 Controller
helm install opensandbox-controller \
  https://github.com/alibaba/OpenSandbox/releases/download/helm/opensandbox-controller/0.1.0/opensandbox-controller-0.1.0.tgz \
  --namespace opensandbox-system \
  --create-namespace
  
# 自定义镜像
helm install opensandbox-controller \
  https://github.com/alibaba/OpenSandbox/releases/download/helm/opensandbox-controller/0.1.0/opensandbox-controller-0.1.0.tgz \
  --set controller.image.repository=<your-registry>/controller \
  --set controller.image.tag=v0.0.1 \
  --namespace opensandbox-system --create-namespace
Shell 
# 构建镜像
cd kubernetes
COMPONENT=controller TAG=v0.0.1 ./build.sh
COMPONENT=task-executor TAG=v0.0.1 ./build.sh

# 从本地 Chart 安装
helm install opensandbox-controller ./charts/opensandbox-controller \
  --set controller.image.repository=<your-registry>/controller \
  --set controller.image.tag=v0.0.1 \
  --namespace opensandbox-system --create-namespace
Shell 
# 构建 + 安装 CRD + 部署
cd kubernetes
make docker-build docker-push CONTROLLER_IMG=<registry>/controller:tag
make docker-build-task-executor docker-push-task-executor TASK_EXECUTOR_IMG=<registry>/task-executor:tag
make install    # 安装 CRD
make deploy CONTROLLER_IMG=<registry>/controller:tag
Shell 
# Controller + Server 一键安装
cd kubernetes/charts
helm dependency build opensandbox
helm install opensandbox ./opensandbox \
  --namespace opensandbox-system --create-namespace
✅ 验证安装
kubectl get pods -n opensandbox-system  |  kubectl get crd | grep opensandbox

CRDs

📜 CRD 安装

三种自定义资源，Helm 自动安装

📦
BatchSandbox
sandbox.opensandbox.io/v1alpha1
  • 沙箱 Pod 组 + 可选 Task Workload
  • 支持 Pool 模式 (poolRef) 和非 Pool 模式 (template)
  • 声明式 Pause/Resume (spec.pause)
🔥
Pool
sandbox.opensandbox.io/v1alpha1
  • 预热 Pod 池 · 四参数容量模型
  • 自动扩缩 + 滚动更新
  • RecycleStrategy: Delete / Restart / Noop
📸
SandboxSnapshot
sandbox.opensandbox.io/v1alpha1
  • 原子快照操作 · Pending → Committing → Succeed
  • Rootfs → OCI 镜像 · commit Job
  • OwnerRef 级联清理
⚙️ 安装方式
Helm 默认自动安装 (crds.install: true)。手动安装: make install (Kustomize) 或 kubectl apply -f kubernetes/config/crd/bases/
卸载保留: crds.keep: true → 添加 helm.sh/resource-policy: keep 注解
Shell 
# 手动删除 CRD (会级联删除所有相关资源)
kubectl delete crd batchsandboxes.sandbox.opensandbox.io
kubectl delete crd pools.sandbox.opensandbox.io
kubectl delete crd sandboxsnapshots.sandbox.opensandbox.io
Server Deployment

🖥️ Server 部署

安装生命周期 API Server + 可选 Ingress Gateway

基本安装

Shell 
helm install opensandbox-server ./kubernetes/charts/opensandbox-server \
  --namespace opensandbox-system --create-namespace

带 Ingress Gateway

Shell 
helm install opensandbox-server ./kubernetes/charts/opensandbox-server \
  --namespace opensandbox-system --create-namespace \
  --set server.gateway.enabled=true \
  --set server.gateway.host=gateway.example.com \
  --set server.gateway.gatewayRouteMode=header

设置 API Key (生产必须)

Shell 
helm install opensandbox-server ./kubernetes/charts/opensandbox-server \
  --set server.env[0].name=OPENSANDBOX_SERVER_API_KEY \
  --set server.env[0].value="your-secret-key"

Port-Forward 本地访问

Shell 
kubectl port-forward -n opensandbox-system svc/opensandbox-server 8080:80

Snapshot Registry

📸 快照 Registry 配置

启用 Pause/Resume 功能需要 OCI Registry

创建 Push/Pull Secret

Shell 
# Push secret
kubectl create secret docker-registry registry-snapshot-push-secret \
  --docker-server=<your-registry> \
  --docker-username=<user> \
  --docker-password=<token>

# Pull secret
kubectl create secret docker-registry registry-pull-secret \
  --docker-server=<your-registry> \
  --docker-username=<user> \
  --docker-password=<token>

Controller 配置

YAML 
controller:
  snapshot:
    registry: "<your-registry>/sandboxes"
    registryInsecure: true   # HTTP / 自签名 Registry
    snapshotPushSecret: "registry-snapshot-push-secret"
    resumePullSecret: "registry-pull-secret"
⚠️ 安全提醒
Commit Job 挂载 Host containerd socket，生产环境必须 pin image-committer-image by digest，并限制 Trusted Registries。
删除 SandboxSnapshot CR 不会 删除已推送的 OCI 镜像，需单独管理镜像生命周期。

Samples

📝 示例资源

快速创建 Pool 和 BatchSandbox

🔥
创建 Pool
YAML 
apiVersion: sandbox.opensandbox.io/v1alpha1
kind: Pool
metadata:
  name: example-pool
  namespace: opensandbox
spec:
  template:
    spec:
      containers:
      - name: sandbox
        image: nginx:latest
  capacitySpec:
    bufferMax: 10
    bufferMin: 2
    poolMax: 20
    poolMin: 5
📦
创建 BatchSandbox (Pooled + Task)
YAML 
apiVersion: sandbox.opensandbox.io/v1alpha1
kind: BatchSandbox
metadata:
  name: my-sandbox
  namespace: opensandbox
spec:
  poolRef: example-pool        # 使用 Pool
  replicas: 3
  taskTemplate:               # 可选
    spec:
      process:
        command: ["sleep"]
        args: ["infinity"]
💡 监控命令
kubectl get pools -A   kubectl get batchsandboxes -A   kubectl describe pool example-pool

Troubleshooting

🔧 故障排查

Shell 
# Controller 日志
kubectl logs -n opensandbox-system -l control-plane=controller-manager -f

# Pod 详情
kubectl describe pod -n opensandbox-system -l control-plane=controller-manager

# CRD 验证
kubectl get crd batchsandboxes.sandbox.opensandbox.io -o yaml

# 分配注解
kubectl get batchsandbox <name> -n <ns> \
  -o jsonpath='{.metadata.annotations.sandbox\.opensandbox\.io/alloc-status}'

# RBAC 检查
kubectl auth can-i --as=system:serviceaccount:opensandbox-system:<sa> create pods

# Chart 验证
make helm-lint
make helm-template-debug

# Kind 集群管理
kind get clusters
kind delete cluster --name sandbox-k8s-test-e2e
⚠️ 常见约束
· Pause/Resume 仅支持 replicas=1 的 BatchSandbox
· make deploy 只更新 Controller 镜像，Task Executor 需单独发布
· Server API Key 为空时需设置 OPENSANDBOX_INSECURE_SERVER=YES (非交互环境)
· Snapshot 镜像生命周期由 Registry 管理，不随 CR 删除
