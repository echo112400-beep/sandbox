Core Concepts

🧠 核心概念

先理清两层 Pool、选池机制和数据流

🏗️
两层 Pool
独立但可桥接
服务端 Pool CRD客户端 SDK Pool
位置Kubernetes CRD + Controlleropensandbox.pool_async
作用预创建 Pod · 亚秒分配本地 ID 缓冲 · 复用连接
管理kubectl / Server APIPython SDK
桥接extensions={"poolRef":"name"}
🎯
自动选池五维 Predicate
  • Capacity — PoolMax - Allocated ≥ replicas
  • Image — Pool 任一镜像 ∈ Sandbox 镜像集
  • LabelSelector — 自定义标签匹配 (需 Profile 启用)
  • NodeSelector — 节点亲和性兼容
  • Resource — CPU/Memory 资源充足
  • Scorer: LeastAllocated (默认) — 选最空闲的池
💡 数据流: SDK extensions={"poolRef":"name"} → Server 写入 BatchSandbox.spec.poolRef → Pool Controller 分配预热 Pod → 沙箱就绪
Step 1 — Create Pools

🏗️ 创建多镜像 Pool 资源池

三种创建方式，适配不同场景

YAML — pool-python.yaml 
apiVersion: sandbox.opensandbox.io/v1alpha1
kind: Pool
metadata:
  name: pool-python
  namespace: opensandbox
  labels:                            # 可选: 用于 LabelSelector 路由
    runtime: "python"
    tier: "standard"
spec:
  template:
    spec:
      containers:
      - name: sandbox
        image: registry.example.com/code-interpreter:v2-python3.12
        command: ["sleep", "infinity"]
        resources:
          limits:   { cpu: "1", memory: "2Gi" }
          requests: { cpu: "500m", memory: "1Gi" }
  capacitySpec:
    bufferMax: 10    # 空闲缓冲上限
    bufferMin: 3     # 空闲缓冲下限
    poolMax: 50     # 池绝对上限
    poolMin: 5      # 池绝对下限
  recycleStrategy:
    type: Restart               # Pod 回池时重启容器
YAML — pool-nodejs.yaml 
apiVersion: sandbox.opensandbox.io/v1alpha1
kind: Pool
metadata:
  name: pool-nodejs
  namespace: opensandbox
  labels: { runtime: "nodejs", tier: "standard" }
spec:
  template:
    spec:
      containers:
      - name: sandbox
        image: registry.example.com/code-interpreter:v2-node20
        command: ["sleep", "infinity"]
        resources:
          limits:   { cpu: "1", memory: "2Gi" }
          requests: { cpu: "500m", memory: "1Gi" }
  capacitySpec:
    bufferMax: 8
    bufferMin: 2
    poolMax: 30
    poolMin: 3
YAML — pool-gpu.yaml (GPU 池) 
apiVersion: sandbox.opensandbox.io/v1alpha1
kind: Pool
metadata:
  name: pool-python-gpu
  namespace: opensandbox
  labels: { runtime: "python", tier: "gpu" }
spec:
  template:
    spec:
      containers:
      - name: sandbox
        image: registry.example.com/code-interpreter:v2-python3.12-gpu
        command: ["sleep", "infinity"]
        resources:
          limits:   { cpu: "4", memory: "16Gi", nvidia.com/gpu: "1" }
          requests: { cpu: "2", memory: "8Gi", nvidia.com/gpu: "1" }
      nodeSelector:                    # GPU 节点亲和性
        accelerator: nvidia
  capacitySpec:
    bufferMax: 3
    bufferMin: 1
    poolMax: 8
    poolMin: 1
🚀 一键部署: kubectl apply -f pool-python.yaml -f pool-nodejs.yaml -f pool-gpu.yaml

📊 容量参数速查: poolMin ≤ total ≤ poolMax · bufferMin ≤ (total-allocated) ≤ bufferMax · 缓冲超范围时回到 (min+max)/2
Step 2 — Use Warm Pods

🔥 使用预热 Pod

两种 SDK 路径：直接指定 poolRef 或通过客户端 Pool 间接使用

🎯
路径 A — 直接指定 poolRef
Sandbox.create() · 最简用法
Python 
from opensandbox import Sandbox
from datetime import timedelta

sandbox = await Sandbox.create(
    # image 可选 — Pool 模板定义镜像
    extensions={"poolRef": "pool-python"},  # ← 关键
    timeout=timedelta(minutes=30),

    # 可选: 覆盖 entrypoint/env
    entrypoint=["python", "-c", "import sys; print(sys.version)"],
    env={"MY_VAR": "hello"},
)

# 使用沙箱
result = await sandbox.command.run("echo hello")
await sandbox.kill()
🔄
路径 B — 客户端 SDK Pool
SandboxPoolAsync · 本地缓冲 + 预热
Python 
from opensandbox import SandboxPoolAsync, PoolCreationSpec, InMemoryAsyncPoolStateStore
from opensandbox.config import ConnectionConfig

spec = PoolCreationSpec(
    extensions={"poolRef": "pool-python"}, # ← 所有创建通过服务端 Pool
    resource={"cpu": "1", "memory": "2Gi"},
)

pool = SandboxPoolAsync(
    pool_name="fast-python-pool",
    max_idle=5,                            # 本地缓冲 5 个就绪沙箱
    state_store=InMemoryAsyncPoolStateStore(),
    connection_config=ConnectionConfig(api_key="...", domain="api.example.com"),
    creation_spec=spec,
)

await pool.start()
sandbox = await pool.acquire(timeout=timedelta(hours=1))
# ... 使用 sandbox ...
await pool.shutdown()
Step 3 — Auto Assign

🎯 自动选池策略

不指定 poolRef 时，Controller 通过 Predicate + Scorer 链自动匹配最优 Pool

🔍
策略 A — 镜像自动匹配
Python 
# 声明镜像需求 → Controller 自动选 pool-nodejs
sandbox = await Sandbox.create(
    extensions={
        "template": {
            "spec": {
                "containers": [{
                    "name": "main",
                    "image": "registry.example.com/code-interpreter:v2-node20"
                }]
            }
        }
    },
    timeout=timedelta(minutes=30),
)
# → 自动匹配 pool-nodejs (镜像匹配) ✓
匹配逻辑: Pool 任一容器镜像 ∈ Sandbox 模板镜像集合 → 通过 ImagePredicate
🏷️
策略 B — GPU 资源过滤
Python 
# GPU 镜像 + GPU 资源 → ResourcePredicate 自动筛掉 CPU 池
sandbox = await Sandbox.create(
    extensions={
        "template": {
            "spec": {
                "containers": [{
                    "name": "main",
                    "image": "registry.example.com/code-interpreter:v2-python3.12-gpu",
                    "resources": {
                        "requests": {"nvidia.com/gpu": "1"}
                    }
                }]
            }
        }
    },
    timeout=timedelta(minutes=30),
)
# pool-python     ✗ (无 GPU 资源)
# pool-python-gpu ✓ (有 GPU 资源)
# pool-nodejs     ✗ (镜像不匹配 + 无 GPU)

Best Practices

🐍 SDK 最佳实践

生产环境推荐模式和完整代码示例

初始化服务端 Pool (一次性)

Shell 
kubectl apply -f pool-python.yaml -f pool-nodejs.yaml -f pool-gpu.yaml
kubectl get pods -l sandbox.opensandbox.io/pool-name   # 查看预热 Pod 状态

创建连接配置 + 客户端 Pool

Python 
from opensandbox.config import ConnectionConfig
from opensandbox import SandboxPoolAsync, PoolCreationSpec, InMemoryAsyncPoolStateStore

# 为每个镜像创建一个客户端 Pool
config = ConnectionConfig(api_key="sk-xxx", domain="api.example.com")

python_pool = SandboxPoolAsync(
    pool_name="python-workers", max_idle=10,
    state_store=InMemoryAsyncPoolStateStore(),
    connection_config=config,
    creation_spec=PoolCreationSpec(
        extensions={"poolRef": "pool-python"},
        resource={"cpu": "1", "memory": "2Gi"},
        env={"PYTHONPATH": "/workspace"},
    ),
    warmup_sandbox_preparer=lambda s: s.files.write_file("/tmp/ready", "ok"),
)

nodejs_pool = SandboxPoolAsync(
    pool_name="nodejs-workers", max_idle=5,
    state_store=InMemoryAsyncPoolStateStore(),
    connection_config=config,
    creation_spec=PoolCreationSpec(
        extensions={"poolRef": "pool-nodejs"},
    ),
)

启动 + 使用时选择对应 Pool

Python 
await python_pool.start()
await nodejs_pool.start()

async def execute_python_code(code: str) -> str:
    sandbox = await python_pool.acquire(timeout=timedelta(minutes=2))
    try:
        result = await sandbox.command.run(f"python -c '{code}'")
        return result.stdout
    finally:
        await python_pool.release(sandbox)  # 归还给客户端缓冲

async def execute_nodejs_code(code: str) -> str:
    sandbox = await nodejs_pool.acquire(timeout=timedelta(minutes=2))
    try:
        result = await sandbox.command.run(f"node -e '{code}'")
        return result.stdout
    finally:
        await nodejs_pool.release(sandbox)

优雅关闭

Python 
import signal

def shutdown_handler(sig, frame):
    asyncio.create_task(python_pool.shutdown())
    asyncio.create_task(nodejs_pool.shutdown())

signal.signal(signal.SIGINT, shutdown_handler)
signal.signal(signal.SIGTERM, shutdown_handler)
Operations

🔧 运维管理

日常运维操作和容量规划

📊
查看 Pool 状态
Shell 
# 列出所有 Pool
kubectl get pools -A
kubectl get pool pool-python -o yaml

# 查看关键指标
kubectl get pool pool-python \
  -o jsonpath='{.status}' | jq
# {"total":15,"allocated":8,"available":5,"updated":12,"revision":"a1b2c3d4"}
📈
动态调整容量
Shell 
# 扩容 — 更新 capacitySpec
kubectl patch pool pool-python --type merge -p '{
  "spec":{"capacitySpec":{
    "bufferMax":20,"bufferMin":5,
    "poolMax":100,"poolMin":10
  }}
}'

# 或通过 Server API
curl -X PUT https://api.example.com/v1/pools/pool-python \
  -H "OPEN-SANDBOX-API-KEY: $KEY" \
  -d '{"capacitySpec":{...}}'
🔄
更新模板 (滚动更新)
Shell 
# 修改镜像版本 → Controller 自动滚动更新
kubectl patch pool pool-python --type merge -p '{
  "spec":{"template":{"spec":{"containers":[{
    "name":"sandbox",
    "image":"registry.example.com/code-interpreter:v3"
  }]}}}
}'

# 查看滚动进度
kubectl get pool pool-python \
  -o jsonpath='{.status.updated}/{.status.total}'
📊 容量规划建议: bufferMin = max_concurrent_users × 0.5 · bufferMax = bufferMin × 3 · poolMin = bufferMin + avg_allocated · poolMax = poolMin × 3
Troubleshooting

🔍 故障排查

问题排查命令解决方法
Pool Pod 不创建 kubectl describe pool <name> · kubectl logs -l control-plane=controller-manager 检查 capacitySpec 是否合法 · Controller 日志 · 节点资源是否充足
Sandbox 长时间 Pending kubectl get batchsandbox <id> -o yaml Pool 无空闲 Pod → 增大 bufferMin · PoolMax 是否已达上限
poolRef 不生效 检查 SDK 请求 extensions={"poolRef":"name"} 确认 Pool 名称正确 · Pool 存在且 status.available > 0
自动选池失败 kubectl get batchsandbox <id> -o jsonpath='{.status.conditions}' 检查 Predicate 是否满足 · 镜像名是否一致 · 资源是否充足
Pod 分配后未释放 kubectl get batchsandbox <id> -o jsonpath='{.metadata.annotations}' 检查 alloc-release/alloc-released 注解 · Finalizer 是否卡住
⚠️ 约束清单
· Pool 模式下 volumes、credentialProxy、platform 不支持 — 由 Pool 模板定义
· Pause/Resume 仅支持 replicas=1 · 暂停后脱离 Pool
· entrypoint/env 通过 taskTemplate 注入覆盖 Pool 默认值
· 客户端 SDK Pool 独立于服务端 Pool CRD — 仅在 extensions 层面桥接
