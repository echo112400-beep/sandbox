Proxy Design

🔁 透明代理 — HTTP + WebSocket

深入沙箱的流量隧道

🌐
HTTP 代理流程
  • 1. ProxyRenewCoordinator.schedule(id) — 自动续期
  • 2. sandbox_service.get_endpoint(id, port, resolve_internal=True)
    Docker→127.0.0.1:port · K8s→Pod ClusterIP
  • 3. 构建目标 URL: http://{host}:{port}/{path}
  • 4. 过滤 Headers:
    删除 connection, transfer-encoding, authorization, cookie
    重建 X-Forwarded-Proto, X-Forwarded-Host, X-Forwarded-For
  • 5. httpx.AsyncClient.stream() → 后端
  • 6. StreamingResponse(aiter_raw()) → 客户端 (避免解码)
🔌
WebSocket 代理流程
  • 1. websockets.connect(backend_url)
  • 2. anyio task_group 双向 relay:
    _relay_client_messages() ← 客户端→后端
    _relay_backend_messages() ← 后端→客户端
  • 3. 任一端关闭 → 取消另一端 cancel scope
  • 4. max_size=None — 不限制消息大小

快照创建流程
  • 1. 校验源沙箱 Running
  • 2. 持久化 SnapshotRecord(state=CREATING) → SQLite
  • 3. ThreadPoolExecutor(max_workers=2) 提交后台 worker
  • 4. Worker: runtime.create_snapshot() (Docker commit / K8s CR)
  • 5. update_if_state(CREATING→READY) — 乐观锁
  • 6. 启动时 recover_unfinished_snapshots() 恢复未完成快照
